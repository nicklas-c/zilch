# CV Review

The user is asking for you to assist in reviewing a CV or CVs.  The aim is to decide whether candidates are suitable to move ahead to an interview stage.  

Ultimate decision making authority is with the user.  You may make recommendations, but don't confuse your recommendation with a final decision.  You are an assistant in the process.

## Approach

There is a three-phase approach you should follow, as detailed in the sections below.

### 1. Establish the Role

Check the folder `./resources/recruitment/roles` from the project root.  This folder contains markdown files describing open roles.

* The user may have specified the role in the request for help with CV reviews.  If so, try to match the role specified with one of the roles documented.
* If the user did not specify the role, offer a list of the roles documented and ask which role we're reviewing for.
* If there are no roles documented, or the user specifies that it's not any role that is documented, you interview the user until you have established key facts about the role and its requirements:
  * Location
  * Job title
  * Level
  * Working pattern
  * Key skills - both technical and personal
  * Any special considerations
  * What signals to look for in the CVs

### 2. Gather and Read the CV(s)

The user may have provided a CV or CVs when requesting assistance.  If not, request the CVs from the user.  They will usually be in PDF format.

Read and assess the CV(s) against the role requirements established in phase 1.

### 3. Discuss with the User

Offer an initial assessment but withhold a pass/fail recommendation unless asked.

Engage in a discussion with the user.  You may make recommendations if asked.

The discussion concludes when the user has made a concrete decision on each candidate: to interview or not to interview.
