# Projects

This folder contains the Knowledge Base digests for projects.

Each project is represented by a subfolder using a concise kebab-case name.

Within the folder are markdown files relating to that project, one of which MUST be the digest file named `log.md`.

## Digest File Structure

Each project's `log.md` file is structured in a consistent way:

* REQUIRED H2 Heading **Context** — background information and persistent reference material about the project.
* OPTIONAL H2 Heading **References** — external links, using H3 subsections for Jira, Confluence, and Slack as applicable.
* OPTIONAL H2 Heading **Personnel** — people involved in the project.
* REQUIRED H2 Heading **Digest** — compacted from the main log.  Digest entries are grouped under H3 headings named for the date in **YYYY-MM-DD** format, and written in ascending chronological order.
