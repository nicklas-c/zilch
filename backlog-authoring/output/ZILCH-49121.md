# ZILCH-49121 — Address Apdex config drift

**Type:** Task
**Parent:** ZILCH-47400

---

## Description

Apdex monitors exist in DataDog for Merchant services with a `[TF]` prefix, indicating they were originally created and managed by Terraform. However, the Apdex configuration has since been commented out in the `terraform-datadog` repo. The monitors are therefore active in DataDog but unmanaged by Terraform.

The risk: a routine `terraform apply` could silently delete these active monitors without any warning.

Investigate why the configuration was commented out — whether this was intentional (e.g. a known issue with the underlying metric) or accidental — and resolve accordingly:

- If the comment-out was accidental or the underlying issue has since been resolved: restore the Terraform configuration and verify the monitors are correctly managed
- If there was a valid reason to disable them: either fix the underlying issue and restore, or explicitly decommission the monitors and remove the dead config

---

## Acceptance Criteria

- The reason the Apdex configuration was commented out is understood and documented
- Terraform state matches the active Apdex monitors in DataDog (no unmanaged monitors)
- Either the Apdex Terraform configuration has been restored and verified, or the monitors have been explicitly decommissioned and the dead config removed
