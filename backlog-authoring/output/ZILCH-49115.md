# ZILCH-49115 — Define functional metrics for fee-service

**Type:** Task
**Parent:** ZILCH-47400
**Blocks:** ZILCH-49116, ZILCH-49118, ZILCH-49123

---

## Context

`fee-service` currently emits standard infrastructure metrics, largely visible in the [service's APM page](https://app.datadoghq.eu/apm/entity/service%3Afee-service).

Two custom Micrometer metrics are defined in code: `fee.service.calculation.successful.counter` and `fee.service.schedule.not.found.counter`.

No log-based metrics are currently available following the disabling of logging in production: https://app.datadoghq.eu/logs?query=service%3Afee-service%20env%3Aeu-ew1-prod

While operational metrics provide good coverage, we are blind on functional health and need to address this.

## Scope

For the purposes of this task, *functional* metrics are considered to be ones that are specific to the business domain, such as calculated fee values. These represent signals that the service is doing what it should be doing, rather than just that it is running in a healthy state.

- Draft a list of *functional* metrics proposed for the fee-service.
- Get list reviewed by team, either in a short meeting, or asynchronously.
- Finalise list and document in Confluence.
