# ZILCH-49116 — Refactor fee-service logging and log levels to support metrics

**Type:** Task
**Parent:** ZILCH-47400
**Depends on:** ZILCH-49115

---

## Context

`fee-service` appears to be logging at level DEBUG only (log messages with other levels appear to be coming from infrastructure, or LaunchDarkly SDKs, etc.).

Meanwhile, DEBUG logging has been turned off in production as a cost optimisation: the service was generating a very high log volume. The effect is that we do not have visibility over key functional metrics for the service.

## Scope

- Review existing logging in `fee-service`.
  - Messages should be logged at an appropriate level, not just at DEBUG by default.
  - Adjust any log levels as appropriate.
- Consider the agreed list of functional metrics for `fee-service`. (See [ZILCH-49115](https://payzilch.atlassian.net/browse/ZILCH-49115))
  - Any logs needed for those metrics should be logging at level INFO or above.
  - Any additional logging required to support the list of agreed metrics should be introduced.
