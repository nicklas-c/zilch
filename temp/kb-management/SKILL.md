description: >-
  Provides instructions for managing the Knowledge Base.
  
  The knowledge base is used to log journal-like entries in order to build context and narrative over time, especially relating to specific well-defined entites such as people, projects, and incidents.  The Knowledge Base is written to incrementally alongside daily activities, noting things as they happen.  Reading back from the knowledge base allows you to assist the user in answering questions such as "what have I achieved this week?", "where do we stand with this project?", "what's the background on this incident?", or "what's relevant to this person?".

  Use this skill to write to the Knowledge Base when:
  - The user's comment notes something happened, for example a meeting or conversation.
  - The user's comment notes a significant decision.
  - The user's comment notes an observation, especially if it's an observation about a known entity, such as a known person, project or incident.
  - On completion of a task, either at the conclusion of a conversation in which you're assisting in a task, or when the user remarks that a task is complete.
  - The user explicitly asks you to write to the log.
  
  Use this skill to read from the Knowledge Base when:
  - The user's request would be supported by knowing the background, context, or status of an entity such as a person, project, or incident.
  - The user is asking to help prepare for a meeting or a 1:1 (one-to-one).
  - The user asks for a summary or status on a project or incident.
  - The user explicitly asks you to read from the log.
